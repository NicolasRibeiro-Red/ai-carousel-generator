## Overview

This design system uses **Tailwind CSS v4** with CSS Custom Properties (CSS Variables) for theming.
All tokens are defined in YAML format and **automatically compiled** to CSS variables via a build pipeline.

## Token Pipeline

```
src/design-tokens/tokens.yaml    ← Single source of truth (DTCG format)
        ↓  node scripts/build-tokens.mjs
src/design-tokens/generated-tokens.css  ← AUTO-GENERATED (do not edit)
        ↓  @import in globals.css
src/app/globals.css              ← Imports generated tokens + @theme inline mapping
```

### Commands

| Command | Description |
|---------|-------------|
| `npm run tokens:build` | Rebuild CSS from YAML |
| `npm run dev` | Auto-rebuilds tokens before starting dev server |
| `npm run build` | Auto-rebuilds tokens before production build |

### Workflow

1. Edit `src/design-tokens/tokens.yaml`
2. Run `npm run tokens:build` (or just `npm run dev`)
3. CSS variables in `generated-tokens.css` update automatically

## Border Radius

Uses CSS variable `--radius` (0.75rem = 12px) as base:
- `rounded-sm` = radius - 4px (8px)
- `rounded-md` = radius - 2px (10px)
- `rounded-lg` = radius (12px)
- `rounded-xl` = radius + 4px (16px)
- `rounded-2xl` = radius + 8px (20px)

## Dark Mode

Dark mode is automatic via CSS variables. Use the `.dark` class on a parent element
or use Tailwind's `dark:` variant for component-specific overrides.

## Canvas/Export Colors

Note: Colors in `lib/constants.ts` (THEME_COLORS, TWITTER_BLUE) remain as hex values
because the Canvas API doesn't support CSS variables. These are used for image export only.
