## Overview

This design system uses **Tailwind CSS v4** with CSS Custom Properties (CSS Variables) for theming.
All tokens are defined in YAML format and compiled to CSS variables.

## Token Structure

```
src/design-tokens/
├── tokens.yaml          # Source of truth (DTCG format)
└── README.md            # This file

src/app/
└── globals.css          # Compiled CSS variables
```

## Spacing Scale

Standard Tailwind spacing scale is used. Semantic spacing can be added to tokens.yaml if needed.

## Border Radius

Uses CSS variable `--radius` (0.625rem = 10px) as base:
- `rounded-sm` = radius - 4px
- `rounded-md` = radius - 2px
- `rounded-lg` = radius (default)
- `rounded-xl` = radius + 4px
- `rounded-2xl` = radius + 8px

## Dark Mode

Dark mode is automatic via CSS variables. Use the `.dark` class on a parent element
or use Tailwind's `dark:` variant for component-specific overrides.

## Canvas/Export Colors

Note: Colors in `lib/constants.ts` (THEME_COLORS, TWITTER_BLUE) remain as hex values
because the Canvas API doesn't support CSS variables. These are used for image export only.
